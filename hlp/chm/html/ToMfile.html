Exporting to m-file

MattPlot can generate a MatLab® m-file (seems to work in FreeMat, may work with minor changes also in Octave and Scilab) containing instruction to import in the math interpreter workspace the tracks data contained in current Frame. The m-file content depends on program version, for actual usage instructions open the file with a text editor and read its comments.

A look to examples section may be useful.

Variable names are taken from track names, converted when necessary to valid identifiers (for example, spaces are converted to underscore). In rare cases two distinct variables may get the same identifier (ex, if a track is named y z and another y_z); when a name clashing occurs the last variable overwrites the previous ones.
