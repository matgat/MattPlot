1) General problems
1.1) When running MattPlot appear a lot of message errors, or program crashes.
Wrong option values may cause errors, try to run program with the default options deleting the option file.
1.2) Something is wrong, the program does not work: what happened?
MattPlot keeps a log of his activity: trigger  ShowLog action and read its lines to understand what happened.
1.3) In my system the MattPlot help doesn't work.
Maybe something is wrong with your HtmlHelp installation. The related files are:
  • %windir%\hh.exe
  • %windir%\system32\hhctrl.ocx
  • %windir%\system32\itss.dll
  • %windir%\system32\itircl.dll
These files are considered as system files, so is not easy dealing with them. You can try to remove and reinstall Microsoft® HtmlHelp library doing something like:
 reinstall.bat
@echo off
title Remove and reinstall HH library
rem download the HtmlHelp library distribution
rem 'hhupd.exe' from http://msdn.microsoft.com
rem and place it in the same dir of this file
set prevdir=%CD%
cd %windir%\system32
rem regsvr32 -u itcc.dll
regsvr32 -u hhctrl.ocx
del hhctrl.ocx
del itss.dll
del itircl.dll
cd %prevdir%
hhupd.exe
2) Program arguments
2.1) When using program passing arguments by command line something goes wrong; inspecting the Log the program complains.
See typical errors in directive section and the notes in program arguments section.
3) Files
3.1) I can't modify the option file, or my changes are ignored.
Check the read-only flag and to use a support that allows writing. Remember that the option file has a fixed name and must be in the same directory of the executable; see option file section for further informations.
3.2) I can't open a plot file.
Maybe the file is corrupted, see the log to view the specific cause. The plot files are simply text files, so you can try to perform manual corrections.
3.3) I can't open a compressed file, nor save plot as compressed archive!
Make sure the zlib API DLL zlibwapi.dll is in path or at least in the MattPlot directory. To see your path:

4) Importing issues
4.1) To open a file I drag and drop it in the plot area; why am I experience a different behaviour every time?
Pay attention on the zone you are releasing the file: if you release it on an existing Frame, tracks will be added to it, otherwise a new frame will be created in the drop position.
4.2) In header directive I can set index unit, but when I declare for example y(cm), the parser thinks that I want to plot y respect the missing column cm instead that y must be plotted respect line index measured in cm. What's the trick?
To declare a self contained independent variable you must use the unit braces: y([cm]). Remember that you can declare also a label: y(L[cm]). Take a look at the examples in the related section.
5) Exporting issues
5.1) Launching from MatLab® an exported m-file named x it complains telling: ??? Undefined function or variable 'x'.
Check MatLab® help about m-file valid names; avoid characters like +-*/ in the file name.
6) Viewing plots
6.1) The plot sometimes change its scale forcing to manually restore the desired scale.
Take a look at plot keys description table: some of them involve a fit to current track, so to keep current scale you should avoid them.
6.2) Sometimes, when zooming with mouse left single clicks, the scale resets.
Check the zoom mode mouse clicks events, maybe left button double click cause frame to fit scale respect current track.
6.3) I would like to see point coordinates using  ShowCoords, but it does not work.
First ensure to select the correct Track (see select mode and plot keys). Then try to zoom points: the coordinates are shown only when point distance is greater than the value set in TrackCoordsDist option entry.
7) Options
7.1) After program startup I see in the log a lot of lines like: Value `xxx´ in section `yyy´ is missing, using default: `zzz´. Is there something wrong in my system? How can I fix this?
It is normal: as described in related section, initially no option file MattPlot.ini is present, so defaults are used. You can create a new one triggering  StoreOptions or  EditOptions actions.
7.2) The keyboard shortcut set in Shortcuts.html option entries does not seem to work.
Be sure that the string represent a valid shortcut; refer to key codes table to see the correct key strings.
7.3) Modifying a Font in the option file my changes are ignored.
Ensure to read options using  ReadOptions. If the Font family is not recognized, windows substitutes it with a default; check your option entry in the option file.
