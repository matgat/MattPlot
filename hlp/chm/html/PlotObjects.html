Plot objects

The concepts defined in the following section are doubtlessly strictly related to program implementation.

Frames

The main element of program interface is the plot area, the area in which data are rendered, constituted by a PaintBox Canvas . This Canvas contains one or more  Frames.
The Canvas is the drawing paper where plot graphic objects are rendered.
The Canvas can contain one or more  Frames, arranged in a defined layout.

A  Frame is a rectangular region of a bidimensional cartesian space characterized by a biunique transformation that binds data values to screen positions (the scale, which is normally linear).

A  Frame is a screen representation of a bidimensional physical space; it contains the following graphic objects that enhance this representation:
  •  axes,  grid,  text labels,  markers
And, of course, some other objects that represents sets of coordinates of the physical plane:
  •  Tracks,  legend
Each Frame contains a set of  Tracks.

Tracks

The term  Track indicates a set of ordered pairs of data, univocally bound with pairs of pixel coordinates. For example, considering a real function, its numeric representation will consists of a set of real numbers ordered respect their corresponding abscissa values (also real numbers); in a similar manner a polar function will be represented by a set of complex numbers ordered respect an independent parameter.
A  Track is an ordered set of data pairs representable on screen.

Legend

The legend lists all Tracks contained in Frame; you can show/hide legend using  LegendToggle action. During certain plot modes you can use legend to select Tracks, and can change its position by dragging it with mouse.

Grid and axes

Grid and axes objects aid user to visualize the physical space mapped on screen frame. To modify grid lines use the following actions:  GridToggle,  GridMore,  GridLess.

Markers

A marker is a vertical (or horizontal) line passing through a given value. You can add a marker in the current frame using  AddMarker action, or by the related directive placed in the data file.

Text labels

A text label is a string bound at a certain position. You can add text in the current frame using  AddText action, or by the related directive placed in the data file.

Program state

Some actions require a target object: a lot of them apply on a Frame (like  RemoveFrame,  FitToTracks etc...), other on a specific Track ( MarkPoints,  TrackStatistics, etc...); in order to make easier passing these arguments to actions, are introduced the state variables current frame within plot and current track within each Frame, both visible in the Status bar; these state variables are modified during the interation with plot area; see Viewing section for detailed informations.
