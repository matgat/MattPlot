Program Features

Some program features are:
  • Installer not required: all is comprised in a single executable, don't need a setup nor particular third party stuff
  • Does not write anything on Windows® registry, so does not require an uninstaller
  • Supports command line arguments, user can choose if create a new instance or use an existing one
  • Graphic interface includes Select, Edit, Pan, Zoom, Measure and Pen tools
  • Plots can be organized in more Frames, allowing data comparison
    • Frames can have different scales
    • Frames layout is adjustable with drag and drop
    • Frames visualization can be synchronized ( LockX,  LockY)
  • Plots can be exported to:
    • Text format
    • MatLab® m-file
    • Bitmap formats (.gif, .bmp)
    • Vectorial image (.emf, .svg)
  • Supported compression formats (see here here for more info and required DLLs):
    • zip archives (*.zip)
    • archives (*.sqx)
    • gzipped files (*.gz)
  • Flexible text data parser
    • can handle columns having different size
    • can perform some data processing like rescale/offset, split, reorder/resample
  • Ehmm, see todo list

Limitations

Some program limitations are:
  • Only 2-d graphs
  • Values are internally represented by IEEE 754 double precision (64bit,15-digit precision) floating point numbers, whose range is about 2.23E-308<|X|<1.79E308; exceeding this range causes math errors
  • Indexes are represented by 32bit signed integers, so data files larger than 2Gb are not supported
  • Does not support Unicode (using char)

On system perturbation

As any program, MattPlot interacts with your system; here's a list of what could happen when you run it.
  • Clipboard overwrite, typically when requesting  Snapshot
  • Media write requests, as specified in the next section

Further notes on storage media access

By developer choice, MattPlot does not write anything in Windows® registry, so handy things like shell integration or file type association are up to users. The program may request to write temporary data in the temporary directory set in TempDir option (the default should be in the system temporary directory %TEMP%\~MattPlot\); generally the deletion of these files is up to system maintainance. Informations that must last over one execution time are stored in MattPlot.ini file, placed in program directory. Summarizing, the reasons that lead MattPlot to request write access to storage media are:
  • On request to edit or store options
  • On request of help file, creation of MattPlot.chm, if missing, in the temporary directory
  • On opening compressed data, to decompress them in temporary directory
  • On crash, creation of a *.log files in the working directory
  • On  Snapshot action
  • On file save user request
  • On starting and on closing, MattPlot may try to delete some temporary files
