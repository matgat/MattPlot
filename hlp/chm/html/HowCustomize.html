How it works

Informations that must last over one execution time (you can call them options) are stored in .ini files.

Reading options

You can force MattPlot to read an option .ini file opening it with  Open action or by drag-and-drop on program window.
As you will see in the next section, there is a special .ini file placed in program directory having the fixed name MattPlot.ini, read at each program execution.

Inspect options

The best way to get familiar with MattPlot costumization is inspecting and editing manually the option file: trigger the  EditOptions action.
Initally no option file is present: MattPlot tries to read it but fails, and in the end it uses its embedded default values. User can create the option file triggering  StoreOptions or  EditOptions actions; the obtained file can then be edited, customizing program. Refer to options table to see available options.

Customization examples

You can try some customization examples choosing  style actions;
  • Change style triggering for example  SetStylePSpice
  • Overwrite the option file triggering  StoreOptions
  • Now analyze and edit MattPlot.ini

Restoring a default option file

In case of damaged or incomplete option file you can restore it following these steps:
  • Close all instances of MattPlot
  • Launch MattPlot
  • Delete current MattPlot.ini file
  • Activate  StoreOptions action
  • MattPlot.ini is restored
If you want to recreate a default option repeat the previous steps removing the file MattPlot.ini before launching program, or alternatively use  SetDefaults and  StoreOptions actions.
