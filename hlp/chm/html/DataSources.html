Data sources

MattPlot uses the concept of stream to interface with external data sources. This will allows to treat strings, files, compressed files, serial line data or any other source of bytes in the same manner.

Files

The main data source is constituted by data previously stored in a medium: the files. You can tell MattPlot to consider a file (with this term I mean also directories) using one of the following ways:
  • Pass its path as program parameter when launching MattPlot
  • Drag and drop it in the plot area; pay attention on the zone you release it: if you release the file on an existing Frame, the tracks will be added in it, otherwise a new Frame will be created in the drop position
  • Using  Open action
The following figure roughly depicts the program behaviour when a file is considered:

If opened file is a supported archive, MattPlot will extract it in the temporary folder (TempDir) and then will try to read the resulting directory.

When opened file is a directory MattPlot will parse recursively all contained files (except those that match ExcludePattern); note that in this case eventual appended lines argument will be used in all these files.

Finally, if opened file is a known type, it will be digested.
File extension Operation Remarks
*.mpx MattPlot will clear and load XML plot Not yet supported
*.ini MattPlot will try to read its options This can be used to manage a set of styles
Compressed files (*.gz) MattPlot will try to read compressed stream Second extension is important (ex. data.txt.gz). Proper DLL must be in path
Compressed archives (*.zip) MattPlot will expand compressed archive and read extracted files Proper DLL must be in path
other extensions (*.txt, *.mpl, ...) MattPlot will try to import as text data file
Generally reading files is a time consuming task: remember that you can use  Abort if it takes too much time.
Note that the extension .mpl is treated as .txt; this strange extension is introduced to provide a file filter and eventually to associate MattPlot with it (start executable by a double click on file).
"mpl" may stand for: MattPLot text file or better: MultiPLe session text file.
Supported compressed format are:
  • *.zip and *.gz with zlib API DLL zlibwapi.dll
