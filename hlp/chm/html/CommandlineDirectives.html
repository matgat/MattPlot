Directives in command line arguments

Directives can be embedded directly in the data file or, in case of need, temporarily appended using program arguments passed through command line. You can append additional lines at the top of a file that is going to be opened passing them before the file path argument, for example, given a file:
 data.dat
 00  10  20  30
 02  12  22  32
Calling program in this way: 
C:\> MattPlot "+#header <a><b><c><d>" "+#tolerant true" data.dat
Has the same effect of opening this file:
 data.dat
#header <a><b><c><d>
#tolerant true
 00  10  20  30
 02  12  22  32

Typical errors

Now a survey of typical errors. Sequence error: 
C:\> MattPlot data.dat "+#header <a><b><c><d>" "+#tolerant true"
C:\> ??ERROR: no file to append directives

C:\> MattPlot "+#header <a><b><c><d>" "+#tolerant true" data.dat
C:\> Ok
Space in arguments error: 
C:\> MattPlot +#header <a><b><c><d> +#tolerant true data.dat
C:\> ??ERROR: what are `<a><b><c><d>´ and `true´?

C:\> MattPlot +"#header <a><b><c><d>" +"#tolerant true" data.dat
C:\> Ok
