System requirements

  • A PentiumPro compatible platform
  • A suitable amount of RAM (see below)
  • A storage media with R/W rights where put executable file
  • A Windows® environment

On memory occupation

During program execution plotted data are permanently buffered in RAM; this imply a dependency of memory occupation on the amount of handled data. To describe how much memory is consumed by data, consider this typical use:
  •  Open a text file containing ten thousands pairs of numbers; assuming string literals of 34 characters per number, in the worst case of file complete buffering, are needed about 700 Kbyte of RAM (once parser had done its work, this memory can be freed).
  • Next we need to store numbers: using 64 bit floating point type we need about 160 Kbyte for the data binary representation, which may increase to 240 Kbyte if stored as complex.
  • Finally, when performing calculations, data container may be temporarily copied doubling the actual used RAM.

Now let's talk about the memory needed by the program itself. The actual code is probably a negligible part respect linked libraries (VCL classes, standard library, etc...); total code will easily reach few Mbyte. Moreover, for performance reasons, the graphic interface is buffered: for example, considering a 800x600 form, we require 800x600x3 byte ~= 1.4 Mbyte of space for the bitmap buffer used to avoid flicker during the repaint.

The conclusion is that the data size become comparable with program size when we deal with at least 100000÷200000 points (pairs of double).
