Transforming tracks

Please that this functionality is evolving and not yet completed. This section consists in just some notes... The tool  TrackTransform gives the user the possibility to create new tracks from an existing ones.

Track operators

A track operator is something that transform one or more track in another one. Available operators are classified by their number of arguments:
  • Unary: applies on a track only, like square root or filters
  • Binary: applies on a pair of tracks, like addition operator
Operator Example Description Note
Unary operators
Product and summation with a scalar m*y + q, m,q scalar Linear transformation Negate: m=-1,q=0
Absolute value |y| Take the absolute value of each sample
Power elevation pow(y,n) Perform n-esimal power elevation of each sample When n<0 and y=0, or n not integer and y<0, a NaN value is given. Square root: n=0.5
Natural logarithm log(y) Perform logarithm (base e) of each sample When argument is null or negative, a NaN value is given
Low pass filter filt(y,0.5) Apply a low pass filter (which? order?) Second argument (0÷1) is the cut-off frequency normalized respect Nyquist freq
Derivative D(y) Calculate the incremental ratios for each pair of samples
Integration S(y,0) Sum the samples area Second argument is the integration constant
Binary operators
Addition y1 + y2 Sample to sample addition Not commutative: resulting track has the first track domain; values of second track corresponding to abscissa of first track are obtained by linear interpolation
Subtraction y1 - y2 Sample to sample subtraction Same
Multiplication y1 * y2 Sample to sample multiplication Same; has precedence respect addition and subtraction
Division y1 / y2 Sample to sample division Same
